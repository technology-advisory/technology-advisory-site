Controles y evidencias: el hardening que se aplicó una vez y nunca se verificó de nuevo

Un servidor endurecido el día de su creación puede desviarse silenciosamente del baseline con cada cambio posterior. La evidencia real es la verificación continua, no la configuración inicial.

1. El hardening que se aplicó una vez, hace tiempo

Un servidor endurecido correctamente en el momento de su creación no permanece endurecido para siempre. Cada cambio posterior —una actualización, una instalación de software, un ajuste de configuración manual— puede desviar el sistema del baseline de seguridad original, sin que nadie lo note hasta una auditoría o un incidente.

2. Baseline documentado con versión, no una configuración implícita

El baseline de hardening aplicado debería estar documentado explícitamente —qué CIS Benchmark, qué versión, qué excepciones justificadas— de forma que sea posible comparar el estado actual contra ese documento de referencia en cualquier momento.

3. Escaneo continuo de configuración, no solo al desplegar

Herramientas de compliance scanning (OpenSCAP, CIS-CAT, o el motor nativo de tu plataforma cloud) ejecutadas periódicamente —no solo en el momento del despliegue inicial— detectan desviaciones del baseline con el tiempo, antes de que se acumulen hasta convertirse en un riesgo significativo.

4. Drift de seguridad: detectarlo antes que un atacante lo explote

El drift de configuración de seguridad (un puerto que alguien abrió "temporalmente", un servicio que se reactivó tras una actualización) es exactamente el tipo de desviación silenciosa que un atacante busca activamente durante el reconocimiento. Detectarlo de forma proactiva es mucho más barato que descubrirlo durante la investigación de un incidente.

5. La evidencia que demuestra diseño, no solo intención documentada

La diferencia entre Security by Design real y Security by Design de papel es la evidencia de verificación continua — no basta con mostrar el documento de la política de hardening, hay que mostrar el resultado del último escaneo de compliance, con fecha y puntuación.
