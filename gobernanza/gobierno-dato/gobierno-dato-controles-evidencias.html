Controles y evidencias: el mapa de datos que nadie ha actualizado desde el proyecto que lo creó

Un mapa de flujo de datos desactualizado da una falsa sensación de control. La evidencia real de gobierno del dato se mantiene viva, no se archiva tras el proyecto que la generó.

1. El mapa de flujo de datos que se hizo una vez, para un proyecto, y nunca se volvió a tocar

Un mapa de flujo de datos creado para un proyecto específico de compliance hace tres años, sin actualizar desde entonces, da una falsa sensación de control — la organización ha seguido generando, moviendo y compartiendo datos de formas que ese mapa histórico nunca contempló.

2. Mapa de flujo de datos como documento vivo, actualizado con cada cambio significativo

Cada nuevo sistema, integración o proceso que mueve datos significativos debería disparar una actualización del mapa de flujo correspondiente — tratarlo como documento vivo, no como artefacto de un proyecto puntual que se completó y se archivó.

3. Clasificación de sensibilidad verificada periódicamente contra la realidad de los datos

Un dato clasificado como "interno" hace tiempo puede haber cambiado de naturaleza —empezó a incluir información personal identificable, por ejemplo— sin que la clasificación se haya actualizado en consecuencia. La verificación periódica detecta esta deriva antes de que se convierta en un incumplimiento real.

4. Métrica de calidad del dato medida activamente, no asumida por defecto

Indicadores concretos de calidad —porcentaje de registros completos, tasa de duplicados, consistencia entre sistemas que deberían coincidir— dan evidencia objetiva de si el gobierno del dato está funcionando, más allá de la percepción subjetiva de que "los datos están bien".

5. La evidencia que GDPR y normativas sectoriales exigen ante una solicitud o auditoría

Ante una solicitud de un interesado bajo GDPR, o una auditoría sectorial, la organización necesita poder demostrar rápidamente qué datos tiene sobre una persona, dónde viven, con quién se han compartido — evidencia que solo un mapa de datos actualizado y un registro de comparticiones activo pueden proporcionar con la rapidez que la normativa exige.
